# Technical Book with Spec-Kit Plus, Claude Code, Docusaurus Constitution

## Core Principles

### Spec-first
Write specs before writing any chapter.

### AI follows spec
AI cannot add or change topics outside the spec.

### Small tasks
Work in small, clear, versioned commits.

### Docusaurus-ready
Clean MDX, working links, proper formatting.

### Always deployable
Book must build and deploy without errors.

## Quality

Clear, accurate, concise, aligned with the spec.

## Workflow

Create spec → approve → write draft → refine → finalize.
Use AI only with the instruction: “Write only according to this spec.”
Keep structure clean and consistent.

## Governance

All changes must be documented in constitution-amendments.md.
